# Typography

## Purpose
Define readable typography for customer and operational screens.

## Business Goals
- Improve scanning, comprehension, and trust.

## Functional Requirements
- Use clear hierarchy for page titles, section headings, labels, body text, captions, and table data.

## User Stories
- As a user, I can scan dense health and order information without strain.

## UI Components
- Headings, labels, helper text, table cells, badges, buttons.

## API Contracts
- Not applicable.

## Zustand Store
- Not applicable.

## React Query Usage
- Not applicable.

## Validation Rules
- Avoid tiny text for critical medical, price, and status information.

## Error States
- Error copy must be direct and readable.

## Empty States
- Empty state titles should be concise.

## Loading States
- Skeleton text lines should match final hierarchy.

## Permissions
- Permission messages must be plain language.

## Future Enhancements
- Responsive type tokens.
