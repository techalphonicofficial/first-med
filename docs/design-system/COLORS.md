# Colors

## Purpose
Define FirstMED color usage.

## Business Goals
- Make the product feel trustworthy, medical, and premium.

## Functional Requirements
- Primary Blue: `#0878BE`
- Primary Yellow: `#F1E300`
- Use neutral surfaces for dashboards and clear semantic colors for states.

## User Stories
- As a user, I can quickly distinguish actions, warnings, success, and errors.

## UI Components
- Buttons, badges, alerts, links, navigation, status indicators.

## API Contracts
- Not applicable.

## Zustand Store
- Theme preference may be stored if introduced.

## React Query Usage
- Not applicable.

## Validation Rules
- Maintain accessible contrast for text and controls.

## Error States
- Error colors should be readable and paired with text.

## Empty States
- Use calm neutral colors with clear CTAs.

## Loading States
- Skeletons should use subtle neutral tones.

## Permissions
- Disabled states must remain understandable.

## Future Enhancements
- Full design token palette.
