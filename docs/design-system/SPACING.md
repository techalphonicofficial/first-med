# Spacing

## Purpose
Define layout rhythm and spacing rules.

## Business Goals
- Keep screens calm, scannable, and professional.

## Functional Requirements
- Use consistent spacing scale for sections, cards, forms, tables, and mobile layouts.

## User Stories
- As a user, I can scan content without visual clutter.

## UI Components
- Page shells, forms, dashboard grids, tables, cards.

## API Contracts
- Not applicable.

## Zustand Store
- Not applicable.

## React Query Usage
- Not applicable.

## Validation Rules
- Avoid cramped tap targets; preserve mobile spacing.

## Error States
- Error text should not shift layouts aggressively.

## Empty States
- Empty states need enough breathing room.

## Loading States
- Skeleton spacing should match loaded UI.

## Permissions
- Hidden actions must not leave awkward gaps.

## Future Enhancements
- Spacing tokens documented from Tailwind config.
